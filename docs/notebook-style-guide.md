# Требования к оформлению учебных блокнотов

Документ фиксирует единый академический стиль Jupyter-блокнотов, используемых в составе пособия.

## Общий принцип

Каждый блокнот должен последовательно раскрывать учебную задачу:

`цель -> данные -> методы -> результаты -> интерпретация -> выводы -> задания`

Блокнот рассматривается как учебный документ, а не как набор несвязанных вычислительных ячеек.
Он должен вести читателя через практическую работу, а не комментировать внутреннюю организацию репозитория.

## Обязательная структура

В каждом основном блокноте должны присутствовать следующие разделы:

1. точное название и краткий подзаголовок;
2. зачем выполняется практикум;
3. что будет сделано и что должен понять читатель;
4. данные и инструменты;
5. этапы работы;
6. код по шагам;
7. интерпретация результатов;
8. итоговые выводы;
9. вопросы или задания для самостоятельной работы;
10. источники.

## Требования к markdown-разметке

- использовать один заголовок уровня `#` в начале;
- основные разделы оформлять заголовками уровня `##`;
- подразделы оформлять заголовками уровня `###`;
- писать в форме учебного сопровождения читателя, а не служебных заметок автора;
- не допускать длинных фрагментов кода без поясняющего текста;
- помещать интерпретацию непосредственно после таблицы, графика или метрики, к которым она относится;
- использовать таблицы для описания библиотек, данных, признаков и метрик.

## Требования к коду

- одна аналитическая операция должна соответствовать одному логическому блоку кода;
- код не должен превращаться в трудно читаемую непрерывную цепочку преобразований;
- промежуточные переменные следует выделять там, где это улучшает понимание хода анализа;
- вывод должен быть управляемым и содержательным.
- в коде и подзаголовках не должно быть внутренней служебной терминологии, не нужной читателю.

## Требования к представлению результатов

Допускается использование:

- `pandas.Styler` для компактных итоговых таблиц;
- `IPython.display.Markdown` и `IPython.display.HTML` для аккуратных поясняющих блоков;
- форматированных таблиц метрик;
- подписанных графиков с последующей текстовой интерпретацией.
- единой стилистики графиков и аккуратной цветовой палитры.

Не допускается:

- декоративное оформление ради визуального эффекта;
- избыточное количество эмодзи и маркеров;
- неструктурированные большие выводы `print`;
- повторяющееся оформление, не добавляющее содержательной ценности.
- повествование в форме внутренней записки разработчику.

## Что заимствуется из шаблона ТюмГУ

Из опорного шаблона заимствуются:

- четкая иерархия разделов;
- логика «данные -> анализ -> интерпретация -> выводы»;
- практикумный ритм из последовательных этапов работы;
- таблицы по библиотекам, источникам и признакам;
- отдельные текстовые блоки интерпретации результатов;
- наличие финальных выводов и списка источников.

## Что исключается

Из учебных блокнотов исключаются:

- лишние декоративные символы в заголовках;
- исследовательские или организационные вставки, не относящиеся к учебной задаче;
- повторы уже сформулированных методических правил;
- любой текст, который не поддерживает освоение конкретного приема анализа данных.
