# Глава 4. Разбиение выборки, метрики и базовые модели

## Цель главы

Объяснить, как корректно разделять данные на выборки, выбирать метрики качества и строить базовые модели сравнения.

После изучения главы читатель должен:

- понимать различие между обучающей, валидационной и тестовой выборками;
- уметь выбирать метрики в зависимости от типа задачи;
- знать, зачем нужна baseline-модель;
- уметь выявлять ошибки, связанные с утечкой данных и некорректной оценкой качества.

## Теоретические сведения

Корректная оценка модели невозможна без дисциплины в организации вычислительного эксперимента.
Даже сильный алгоритм не имеет ценности, если ошибка измерена на тех же данных, по которым модель подбиралась и настраивалась.

Для табличных задач обычно используются:

- `train` — для обучения;
- `validation` — для выбора настроек и сравнения вариантов;
- `test` — для итоговой независимой оценки.

Для временных рядов используется временное разбиение, при котором будущие наблюдения не должны участвовать в формировании признаков и оценке прошлых периодов.

Выбор метрики должен быть связан с содержанием задачи:

- для регрессии — ошибка в физических величинах и относительная чувствительность к крупным отклонениям;
- для классификации — соотношение пропусков и ложных срабатываний;
- для временных рядов — устойчивость качества по горизонту прогноза.

## Ключевые термины

- **обучающая выборка** — часть данных, на которой оцениваются параметры модели;
- **валидационная выборка** — часть данных для сравнения вариантов модели;
- **тестовая выборка** — независимая часть данных для итоговой оценки;
- **baseline-модель** — простая исходная модель, задающая минимальный уровень сравнения;
- **метрика** — количественный показатель качества прогноза или классификации.
