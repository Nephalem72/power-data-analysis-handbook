# Jupyter Environment Notes

Этот документ фиксирует рабочую позицию проекта по состоянию на март 2026 года.

## Основная среда

Для курса основной средой считается **JupyterLab**.

Причина простая: в актуальной официальной документации JupyterLab описывается как next-generation web-based user interface для Project Jupyter и как более гибкая интегрированная рабочая среда с файлами, блокнотами, терминалом и другими документами.

Официальный источник:
[JupyterLab Get Started](https://jupyterlab.readthedocs.io/en/stable/getting_started/overview.html)

## Что делать с классическим Notebook

В курсе не нужно искусственно разводить `JupyterLab` и `Jupyter Notebook` как две разные учебные системы.
Для студента основной сущностью остается `.ipynb`, а интерфейс выбирается по удобству.

При этом корректно писать так:

- `Jupyter Notebook` — notebook authoring application для работы с вычислительными документами;
- `JupyterLab` — более широкая рабочая оболочка поверх тех же базовых сущностей.

Официальный источник:
[Jupyter Notebook docs](https://jupyter-notebook.readthedocs.io/en/stable/notebook.html)

## Практическая формулировка для пособия

Рекомендуемая формулировка в тексте пособия:

> Основная рабочая среда курса — JupyterLab. При необходимости те же блокноты могут открываться в Jupyter Notebook, поскольку учебный материал опирается не на особенности конкретной оболочки, а на формат `.ipynb`, воспроизводимый код, Markdown-разметку и интерпретацию результатов.

## Что важно показать в установочном разделе

- запуск `jupyter lab` и `jupyter notebook`;
- структура `.ipynb`;
- кодовые и markdown-ячейки;
- запуск ячеек и перезапуск ядра;
- сохранение и экспорт;
- связь блокнотов с файлами проекта;
- типовые ошибки: не то окружение, не тот kernel, путь к файлу, не перезапущенное ядро.
