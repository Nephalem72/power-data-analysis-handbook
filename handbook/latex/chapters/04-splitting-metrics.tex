\chapter{Разбиение выборки, метрики и базовые модели}

\section{Зачем требуется разбиение выборки}

Оценка качества модели не имеет смысла, если выполняется на тех же данных, по которым модель настраивалась.
По этой причине в машинном обучении и прикладном анализе данных вводится разбиение выборки на несколько частей.

В простейшем случае используются:
\begin{itemize}[leftmargin=1.25cm]
\item обучающая выборка;
\item тестовая выборка.
\end{itemize}

В более строгой постановке добавляется валидационная выборка, используемая для выбора настроек и сравнения альтернативных моделей.

\section{Базовая модель}

Перед переходом к более содержательной модели требуется baseline.
Baseline-модель задает минимальный уровень, который должна превзойти любая более сложная схема.

Для регрессии такой моделью может быть, например, прогноз средним значением:
\[
\hat{y}_i = \overline{y}_{train}.
\]

Если более сложная модель не улучшает этот ориентир, дальнейшее усложнение не имеет практического смысла.

\section{Метрики качества}

Для задач регрессии в пособии рассматриваются следующие основные метрики.

\subsection*{Средняя абсолютная ошибка}

\[
MAE = \frac{1}{n}\sum_{i=1}^{n} |y_i - \hat{y}_i|.
\]

Данная метрика удобна тем, что выражается в тех же единицах, что и прогнозируемая величина.
В инженерной постановке это облегчает интерпретацию.

\subsection*{Корень из средней квадратичной ошибки}

\[
RMSE = \sqrt{\frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2}.
\]

Метрика сильнее штрафует крупные ошибки и потому чувствительнее к единичным сильным отклонениям.

\subsection*{Коэффициент детерминации}

\[
R^2 = 1 - \frac{\sum_{i=1}^{n}(y_i - \hat{y}_i)^2}{\sum_{i=1}^{n}(y_i - \overline{y})^2}.
\]

Эта метрика удобна для общего сравнения, но не должна использоваться в изоляции от абсолютных ошибок.

\section{Особенности временных рядов}

Для временных рядов случайное разбиение недопустимо, поскольку нарушает естественный порядок наблюдений.
В таких задачах обучение должно выполняться только по прошлым данным, а оценка --- по последующим периодам.
Именно поэтому для временных рядов используются временные схемы разбиения и backtesting.

\section{Что должен сделать обучающийся}

После выполнения главы обучающийся должен уметь:
\begin{itemize}[leftmargin=1.25cm]
\item разделять данные на обучающую и тестовую выборки;
\item строить baseline-модель;
\item рассчитывать базовые метрики качества;
\item обосновывать выбор метрики с точки зрения содержания задачи.
\end{itemize}
